# 009. Pathfinding For Large Groups

## Коротко

Стабилизировать pathfinding для больших групп и узких проходов: меньше repath spikes, меньше stuck, лучше поведение 60-200 юнитов.

## Контекст проекта

Сейчас pathfinding работает для обычных сценариев, но `task/requests.md` отдельно просит стабилизировать группы и проверить лимит `findPath`. После задачи 008 группа уже должна иметь center path и formation slots. Теперь надо уменьшить дорогие пересчеты и улучшить локальный обход.

## Предыдущие задачи

Задача 008 является важным prerequisite: без formation slots pathfinding будет пытаться привести всех в одну точку. Если 008 не сделана, сначала не стоит начинать глубокую оптимизацию pathfinding.

## Что сделать

1. Добавить debug/test scenario:
   - 60 юнитов;
   - узкий проход;
   - одна команда движения через проход.
2. Измерить текущую проблему:
   - сколько раз вызывается `findPath`;
   - сколько занимает средний/пиковый поиск;
   - сколько юнитов stuck через N секунд.
3. Добавить group repath cooldown:
   - не пересчитывать path каждый frame;
   - учитывать group movement state.
4. Добавить path batching/reuse:
   - один path для center;
   - переиспользование сегментов для nearby slots;
   - отдельный path только при явной необходимости.
5. Улучшить local avoidance:
   - обход стоящих союзников;
   - не толкаться в одной клетке;
   - не создавать бесконечный repath loop.
6. Защитить одиночные команды:
   - одиночный юнит должен вести себя как раньше или лучше.

## Implementation notes

- Основные файлы: `src/systems/Pathfinding.ts`, `src/systems/Formation.ts`, `src/scenes/GameScene.ts`.
- Не внедрять полноценный HPA*/flow-field, если batching/cooldown решают основную проблему за одну сессию.
- Логи/метрики можно держать за debug flag.
- Следить, чтобы AI pathfinding тоже не сломался.

## Проверка

- `npm run build`
- Debug scenario 60 юнитов через узкий проход.
- Проверить:
  - нет массового stuck;
  - нет заметных frame spikes;
  - одиночный юнит строит path корректно;
  - attack-move не ломается.

Задача 010 будет смотреть не только pathfinding, но и общий frame time на 100/300/500 юнитах. Результаты этой задачи должны дать ей стабильную основу.
